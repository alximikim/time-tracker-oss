# Сборка и раздача клиента (`client/TimeTracker`)

Клиент — трей-приложение для Windows. Сотрудник видит только запрос имени
при первом запуске — адрес сервера и токен он не вводит и не видит.

## 1. Сборка

Self-contained публикация — один exe, не требует установки .NET на машине
сотрудника.

```powershell
dotnet publish client/TimeTracker -c Release -r win-x64 `
  --self-contained true -p:PublishSingleFile=true `
  -o publish/client
```

Результат — `publish/client/TimeTracker.exe`.

## 2. Подготовка server-config.json

Скопируйте шаблон `docs/server-config.example.json` в `publish/client/server-config.json`
(рядом с exe) и впишите реальные значения — адрес вашего сервера (см.
[`DEPLOY-SERVER.md`](DEPLOY-SERVER.md)) и тот же `SharedToken`, что задан на
сервере:

```json
{
  "apiUrl": "http://<адрес-сервера>:<порт>/api/events",
  "sharedToken": "<тот же SharedToken, что в конфиге сервера>",
  "enableBreakButton": false,
  "longSessionReminderHours": null
}
```

Это единственный шаг, который делает администратор один раз перед раздачей —
все сотрудники получают одинаковую пару файлов.

Два последних поля — необязательные фичи, включаются на уровне всей
организации (не у каждого сотрудника отдельно):

- **`enableBreakButton`** (`true`/`false`, по умолчанию `false`) — добавляет
  в трей-меню кнопку "Перерыв" для обеда/отлучек. Технически это обычная
  пара стоп/старт с отдельной пометкой в статусе — в админ-панели сервера
  будет видно несколько сессий в день на сотрудника вместо одной, итоговые
  часы за день считаются суммой автоматически, ничего вручную досчитывать
  не нужно.
- **`longSessionReminderHours`** (число часов, например `8`, или `null` —
  выключено) — раз в день, когда суммарные отработанные часы (все сессии,
  включая те, что до перерыва) впервые достигают порога, сотруднику
  показывается всплывающее уведомление трея с предложением продолжить или
  закончить работу. Ничего не останавливает и не завершает автоматически —
  только напоминание.

## 3. Раздача сотрудникам

Заархивируйте папку `publish/client/` (в ней `TimeTracker.exe` +
`server-config.json`) и разошлите сотрудникам. Каждый сотрудник:

1. Распаковывает архив в любую папку.
2. Запускает `TimeTracker.exe`.
3. Если `server-config.json` не найден или повреждён — приложение покажет
   ошибку и попросит обратиться к администратору, трей не запустится.
4. При первом успешном запуске появится окно "Введите ваше имя и фамилию" —
   единственное, что видит сотрудник.
5. После сохранения имени приложение уходит в трей и само поднимается при
   каждом входе в Windows (автозапуск через `HKCU\...\Run`).

Сменить сотрудника (например, при передаче ноутбука другому человеку) —
пункт "Сменить сотрудника" в контекстном меню трея.

## 4. Что видит сотрудник дальше

- Иконка в трее: без бейджа — сессия не идёт, зелёный бейдж — сессия
  активна, оранжевый бейдж — перерыв (если `enableBreakButton` включён).
- Кнопки "Начать работу"/"Закончить работу" в меню (или двойной клик по
  иконке — двойной клик всегда работает как старт/стоп, независимо от
  кнопки паузы).
- Если включён `enableBreakButton` — пункт "Перерыв" (во время сессии) /
  "Вернулся с перерыва" (во время паузы).
- Тултип показывает текущее отработанное время сессии.
- Если задан `longSessionReminderHours` — всплывающее уведомление трея,
  когда сотрудник отработал за день заданное число часов.

Никакого автостарта сессии по включению компьютера нет — включённый
компьютер сам по себе не засчитывает время, только явное нажатие кнопки.
Аналогично, перерыв — это тоже явное действие сотрудника, а не
автоматический детект бездействия.
